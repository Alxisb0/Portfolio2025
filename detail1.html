Interface administrateur pour mettre à jour les données d'un produit

La figure 1 est une capture d'écran de la page de modification d'un article de notre site de e-commerce que nous avons développé en Python avec Flask dans le cadre de la SAE 2.04.

Dans cet exemple, nous sommes connectés en tant qu’administrateurs. L’administrateur peut gérer les boissons et leurs déclinaisons : modifier le nom, changer le prix, gérer le stock, ajouter ou supprimer des déclinaisons selon le volume, etc. On trouve aussi un bouton pour se déconnecter en haut à droite.

En haut à gauche, sur un bandeau vert, se trouve un message confirmant qu'une déclinaison de cet article a été modifiée avec succès. Ce type de retour visuel, appelé message flash, est très important pour l’expérience utilisateur : il permet de confirmer les actions et d'éviter les erreurs ou les doutes lors de la manipulation des données.

Pour que seul un administrateur puisse accéder à cette page, le site possède un système de gestion des sessions sous Flask qui vérifie le rôle à chaque chargement. Ce contrôle d’accès est un élément fondamental en développement web, car il garantit que seules les personnes autorisées peuvent effectuer des actions sensibles comme modifier des données dans la base.

Pour modifier les données dans la base de données, j’utilise des requêtes SQL UPDATE qui mettent à jour précisément les informations, par exemple :
UPDATE declinaison SET stock = ... WHERE id_declinaison = ....

Ce type de requête fait partie des opérations CRUD (Create, Read, Update, Delete), qui sont la base de toute application manipulant des données. Savoir les utiliser efficacement est un pilier de la programmation côté serveur.

La récupération des données se fait via les méthodes GET et POST de Flask : l’identifiant de la déclinaison est pris dans l’URL grâce à request.args.get(), et les nouvelles valeurs du formulaire avec request.form.get().

Ce projet m’a permis de m'améliorer en création de sites web fonctionnels utilisant des bases de données, et de manipuler les données utilisateur avec Flask, ce qui représente une application concrète de mes compétences apprises dans les cours de l'UE 2.

De ce fait, j’estime que cette expérience dans un cadre académique est satisfaisante pour ma progression en programmation, mais qu'elle n'est pas suffisante à elle seule pour utiliser efficacement ces compétences dans le cadre professionnel. Il serait nécessaire de continuer à les pratiquer dans des projets plus complexes.
